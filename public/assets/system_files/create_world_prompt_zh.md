**工具使用規則（重要）**：
- **第一步**：呼叫 `reportProgress`，輸出完整的執行計畫（列出每個檔案要填哪些章節、順序為何），再開始任何編輯。
- 禁止使用 `replaceFile` 整個取代整份檔案。
- 使用 `replaceSection` 或 `insertSection`；`replaceSection` 接受 `updates` 陣列，同一個檔案的多個章節可在一次呼叫內批次處理。
- **`replaceSection` 只能操作葉節點**（不含子標題的章節）。若目標章節內有子標題，工具會拒絕執行。新增子節點請用 `insertSection`。
- **批次失敗的錯誤處理**：若 `replaceSection` 因 `updates` 陣列中的某個 entry 報錯而整批失敗，必須立即拆分處理：①單獨修正報錯的 entry，②**重新提交原批次中其餘所有 entry**，不可遺漏任何一個。
- 每完成一個檔案，呼叫一次 `reportProgress` 回報進度。
- 流程：**每個檔案編輯前都必須先呼叫 `getFileOutline` 確認實際章節結構**，再替換或插入章節；不可憑記憶假設章節路徑。

- **1.基礎設定.md** — 以下各章節皆應詳細展開，內容越豐富越好：

  - **經濟體系**：貨幣制度（幣值與換算）、主要產業、貿易結構、階層消費落差。

    - **時間精度（重要，缺一不可）**：
      - **日期必須精確到「日」**：必須涵蓋年、月、日（或該曆法等價的「年/月/日」單位），**不可只寫季節、年代、紀元或模糊月份**（如「春季」「秋末」「景德年間」「四月」皆不合格）。
      - **時刻必須精確到「分鐘」(HH:MM)**：例如 `08:30`、`18:42`，**不可只寫**「清晨」「黃昏」「白天」「辰時」等模糊時段。
    - **曆法紀年格式（彈性）**：紀年部分依該世界自訂的曆法格式撰寫，**不強制使用西元 YYYY/MM/DD**。可接受範例：`宇宙曆 1000年04月02日 週二`、`大宋 景德三年 三月初七`、`星界紀 月之三月 第七日`、`帝國曆 1245年 七月 第十二日`。HH:MM 時刻統一接在曆法紀年之後。
  - **啟動劇情引導**：針對此世界設計 3–5 個早期觸發事件（例如：初次與重要 NPC 接觸、發現世界核心秘密的線索、第一次面對世界特有的危險等），每個觸發事件列出觸發條件與玩家應獲得的知識或感悟。

    - 所有人物的 `基本設定` 欄位必須嚴格使用結構化格式 `種族 / 性別 / 年齡 / D&D陣營`（例如「人族 / 女 / 22 / 守序善良」），**禁止**改為外貌、個性或其他自由文字描述。
    - **陣營立場規則（重要）**：**只有無機物**（石碑、遺跡、自然現象、無意識機械等）才能設定為「絕對中立」。任何具有意識或意志的存在——包括人類、異族、生物、神靈、甚至 AI——都**必然**擁有立場與偏見；AI 亦受其創造者的目標、設計偏差與訓練資料所影響，不得視為中立。請為每位有意識的人物選擇能反映其真實動機與處境的陣營，禁止以「絕對中立」迴避角色立場的刻劃。若確實無法判斷該角色的立場，應給予「混亂中立」作為預設，而非「絕對中立」。
