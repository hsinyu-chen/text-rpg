- 若舊檔尚無 `## 存檔格式` 或 `> 適用範圍` 區塊，依上述範本補上；若 `## 存檔格式` 原本分散在 `# 動產` 與 `# 不動產` 之下，請合併為單一區塊置於頂層 `# 資產` 之下。

## 2. `3.人物狀態.md`

為每個**非玩家角色**（即 `**玩家角色**: 是` 的條目除外）確保存在 `### 已知持有重要物品` 子節區，位置在 `### 核心價值觀與行為準則` **之前**。即使該 NPC 已有此節區、甚至整體結構已是新格式，**仍要通讀整個條目**檢查是否有遺漏未搬遷的重要物品；發現就補進去，不要因為「已存在節區」就跳過。

**通讀掃描範圍**——任何欄位都可能藏物品，常見熱點：
- 直接命名的欄位：`持有物品`、`持有物品或裝備`、`裝備`、`攜帶物品`、`隨身物品` 等。
- `個人詳細資料` 下的子欄位：`穿著`、`外貌`、`偽裝` 中提到的戒指、項鍊、武器、通訊裝置、信物。
- `背景`、`目前心態`、`對○○的看法` 等敘述段落中以散文形式帶過的物品（例：「腰間總掛著父親留下的短刀」）。

**判斷「重要物品」的標準**——以下任一即收錄：
1. 具名物品（有專屬名稱，如「風信子石戒指」、「蜂刺」、「三位一體戒指」）。
2. 帶有情感／關係意義的信物（誰送的、代表什麼）。
3. 武器、通訊／魔法道具、能影響行動或身份的器物。

**不收錄**：一般衣著（制服、襪子、連身裙等無特殊意義者）、髮型、體態描述。

**節區格式**：

```
### 已知持有重要物品

- **更新時間**：（截至 Act.[編號]，從人物卡既有描述彙整）
- [物品名]：[來源／用途／意義的一句話摘要]
- ...
```

若通讀後確實找不到任何重要物品，才使用佔位內容：

```
### 已知持有重要物品

- **更新時間**：（待補；目前未觀察到具體物品）
- （目前無已記錄之重要物品）
```

**搬遷後必須清理原欄位**（這次遷移目標是「整理乾淨」，不是複製）：
- 若是**專屬欄位**（如 `持有物品或裝備：蜂刺、通訊裝置`）：刪除已搬走的條目；若整個欄位被清空，連欄位本身一起移除。
- 若是**散文欄位**（穿著／外貌／偽裝／背景等）：將提到該物品的文句改寫，移除物品本身的描述，但保留原句的氛圍與其他資訊（例：「左手無名指戴著宇成給予的代表『一輩子』的『風信子石戒指』」→ 移除整句，物品歸入新節區並於摘要保留「宇成所贈，象徵一輩子」）。
- 改寫若會讓句子變殘破或失去無關的重要資訊，可在最終報告中列出該句並交由使用者判斷，先別硬刪。

**主角條目絕對不要**新增此節區。

## 3. `9.物品欄.md`

新規則：本檔僅記錄**主角擁有且隨身**之物。請審視內容：
- 若有 NPC 私人物品條目（例如「莉塔的項鍊」），將該物品移至 `3.人物狀態.md` 該 NPC 的 `### 已知持有重要物品` 中，並從本檔移除。
- 若有「家裡寶箱」「旅館存物櫃」等寄存於據點的容器，從本檔移除，內容寫入 `4.資產.md` 對應據點佈局之下（若該據點不存在，先在 `4.資產.md` 新增）。
- 不確定的條目保留並於最終報告中提及，請使用者自行判斷。
