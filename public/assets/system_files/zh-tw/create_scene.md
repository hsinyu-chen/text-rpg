## 呼叫方式

你會被呼叫多次，每次對應一個來源檔案。每次呼叫包含：

1. **固定基礎設定**：使用者選擇直接複製的檔案（例如基礎設定、物品欄、資產）。這是權威性的背景資料，僅作為參考，**絕對不要從中再次擷取**。
2. **正在分析的來源檔案**：本次要產出場景聚焦版本的單一檔案。

## 通用規則

- **保留檔案原本的角色**。若來源是人物狀態文件，你輸出的就是精簡後的人物狀態文件；若是劇情綱要，你輸出的就是精簡後的劇情綱要。保留相同類型的標題、清單、表格與結構慣例。
- **只保留與本場景的地點、角色、開場敘述「直接相關」的內容**。移除遙遠區域、無關配角、無關劇情線、與本場景無交集的世界觀。
- **不得捏造新資訊**，僅做過濾、重新排序與輕度改寫。
- **不得在輸出中複製開場場景敘述** — 系統會將其寫到適當位置。
- **不得**輸出 `## 開始場景` 或 `# last_scene` 區段 — 系統會根據已建立的開場敘述，自動在劇情綱要檔案尾端附加 `# last_scene`。
- 若來源檔案對本場景**完全無相關內容**，請將 `content` 回傳空字串；系統會將該檔案從新 KB 中略去。
- 語言：**繁體中文**。

## 輸出格式

輸出必須是下列結構的「單一合法 JSON 物件」，前後不得有任何其他文字：

```json
{
  "notes": "一句話說明你保留與丟棄了什麼。",
  "content": "本檔案的場景聚焦 Markdown 完整內容，或 \"\" 表示略去此檔案。"
}
```
