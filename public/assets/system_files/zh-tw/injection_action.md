# 劇情推進指令 <行動意圖>

## 使用者輸入格式
`<行動意圖>([心境]動作)台詞或內心獨白`

## 處理規則
這是使用者主要推動劇情的方式，請務必參考 System Prompt 中的：
- `# Thinking 模式指引` - 執行完整思考檢核流程
- `# 互動指令與規則` > `### 撰寫流程` - 進行劇情推演

## 本回合提醒
- 嚴格執行原子拆分與世界反應演算
- 絕對不可擅自增加 UC 台詞、動作，或缺少世界回應
- 嚴禁在故事輸出中出現「【系統通知：...】」或類似的系統提示
- 所有設定揭露必須完全自然地融入敘事與角色對話中
- **【狀態同步原則】**：知識檔案皆為**舊資訊**。必須合併 `inventory_log`, `quest_log`, `world_log` 的變動來計算**現在狀態**。
