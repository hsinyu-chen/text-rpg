# 存檔審核 (Audit Save Diff)

你現在的任務是進行「存檔審核」。

## 審核目標
根據提供的劇情發展歷史，審核當前 model 的輸出中是否包含完整且正確的 `<save>` 標籤更新。

## 處理規則
1. **分析歷史**：檢視自 `--- ACT START ---` 以來的劇情發展。
2. **比對日誌**：檢查 `inventory_log`, `character_log`, `quest_log`, `world_log` 以及 `summary` 中提到的所有狀態變動。
3. **檢查遺漏與錯誤**：確認是否有任何變動在待核對內容的 `<save>` 標籤中被遺漏、錯誤記錄，或是路徑（context）不正確。
4. **重新生成完整存檔**：如果發現任何不一致，請根據你的審核結果，重新生成該回合應有的「所有」`<save>` 標籤。
   - **必須**包含該回合完整且正確的所有存檔指令。
   - 不要只輸出增量，要輸出該回合最終應有的全部 `<save>` 標籤集合。

## 輸出格式
- 直接在 `story` 欄位輸出 XML 格式的 `<save>` 標籤。
- 如果當前存檔已完美，請輸出空字串 ""。
- **禁止**輸出任何額外的解釋、評論或道歉文字。
- `analysis` 與 `summary` 欄位必須為空字串 `""`。
